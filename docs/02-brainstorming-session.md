# Этап 2. Мозговой штурм: Социальная сеть для хоккеистов-любителей

## Кластер 1: Боли и проблемы (Pain Points)

### Организация и коммуникация
- **Фрагментация:** Сбор состава в WhatsApp/Telegram, таблицы в Excel, оплата переводом на карту капитану.
- **Поиск «замен»:** Срочный поиск игрока или вратаря (особенно вратаря!) за несколько часов до игры.
- **Дисбаланс уровня:** «Мастера» приходят на игры «Дебютантов» и наоборот. Отсутствие единого рейтинга (Skill Level).
- **Сбор денег:** Проблема «забывчивых» игроков. Капитан превращается в коллектора.

### Лед и локации
- **Слепое бронирование:** Непонятно качество льда, наличие нормальных раздевалок и заточки коньков, пока не приедешь.
- **Прайм-тайм:** Весь хороший лед занят школами или корпоратами.

### Статистика и признание
- **Разрозненность лиг:** Статистика игрока в НХЛ не видна в ЛХЛ. Нет «единого паспорта хоккеиста».
- **Медиа-голод:** Сложно найти фото/видео своих голов или сейвов после матча.

## Кластер 2: Порталы аренды льда (Ice Infrastructure)

### Потенциальные источники данных
1. **НаЛёд (Naled.pro):** Один из крупнейших агрегаторов. Вероятно, имеют внутреннее API для партнеров.
2. **Спорт-Сбор:** Агрегатор тренировок и аренды.
3. **Арена-Маркет:** Система управления ледовыми дворцами.
4. **МосГорСпорт / mos.ru:** Данные о государственных катках.

### Ожидаемые данные (Data Schema)
- `arena_name`, `address`, `location_gps`.
- `slots`: `start_time`, `end_time`, `price`, `status` (free/booked).
- `amenities`: раздевалки, душ, парковка, прокат, заточка.

## Кластер 3: Любительские лиги (League Ecosystem)

### Примеры для интеграции
- **НХЛ (Ночная Хоккейная Лига):** Самая массовая, жесткие регламенты.
- **ЛХЛ-77 / РТХЛ:** Популярные московские лиги.
- **Корпоративные лиги:** (Лига ТЭК, Банковская лига).
- **Региональные:** СПбХЛ (Питер), лиги Татарстана, Урала.

### Что парсить/получать?
- **Rosters:** Списки команд и заявки.
- **Game Sheets:** Протоколы матчей (кто забил, кто удалился).
- **Standings:** Турнирные таблицы.
- **Suspensions:** Дисквалификации (важно для проверки игрока перед добором).

## Кластер 4: Магазины и экипировка (Gear Integration)

### Потребности пользователей
- **Сравнение цен:** Хоккейная форма дорогая, важен поиск лучшей цены.
- **Совместимость:** «Подойдет ли этот крюк под мой стиль игры?».
- **Вторичный рынок:** Безопасная покупка б/у формы (особенно для детей, которые быстро растут).

### Идеи для фич
- **Gear Bag:** Виртуальный баул в профиле. Уведомления: «Твоим конькам 2 года, пора проверить лезвия».
- **Reviews:** Отзывы именно от любителей, а не от про-игроков.

## Кластер 5: Технические возможности и риски

### API vs Scraping
- **Open API:** Почти отсутствуют в нише.
- **Web Scraping:** Основной метод получения данных о лигах и магазинах на старте.
- **Admin Panel:** Ручной ввод данных администраторами арен (нужна сильная мотивация/ценность).

### Гипотезы для проверки (Hypotheses)
1. **H1:** Если дать капитану инструмент «автосбор денег», он приведет всю команду в приложение.
2. **H2:** Если вратари будут получать «карму» за явку, проблема их поиска решится на 40%.
3. **H3:** Интеграция со статистикой лиг повысит Retention (игроки любят смотреть на свои цифры).

## Кластер 6: «Мёртвые» внешние действия в Phase 1 (Mock External Flows)

### Проблема
Кнопки «Записаться на лёд», «Купить», «Сайт лиги/магазина» вели на `example.ru` и визуально ничего не делали: Gravity UI `Button href` не давал понятного UX, а фиктивные URL не открывались в демо-среде.

### Где встречалось
| Сценарий | Было | Риск для демо |
| :--- | :--- | :--- |
| Запись на лёд (арена) | `Button href={bookingUrl}` | Кнопка «мёртвая» |
| Бронирование слота | То же | Нет подтверждения действия |
| Покупка товара | `<a href={externalUrl}>` | Уход на несуществующий сайт |
| Сайт лиги | `<a href>` | Нет контекста интеграции |
| Сайт магазина | `<a href>` | Нет контекста партнёрства |

### Решение Phase 1
- Единая оболочка `MockExternalFlowDialog` с пометкой **Phase 1 mock** и целевым URL Phase 2.
- **Ice booking wizard** — телефон, комментарий, mock-код `ICE-XXXXXX`.
- **Shop checkout** — подтверждение цены и mock-redirect.
- **League/Shop portal preview** — краткое превью партнёра без реального перехода.

### Гипотезы
1. **H4:** Кликабельный mock-flow повышает доверие к демо у капитанов и организаторов.
2. **H5:** Показ целевого URL Phase 2 ускоряет переговоры с аренами и магазинами.
3. **H6:** Единый паттерн mock external flow упростит миграцию на backend adapters.

### Phase 2
- Тот же UI-компонент переключается на `window.open(partnerUrl)` или deep link partner API.
- MSW endpoint `ice-booking-requests` становится proxy к NaLed / Arena-Market adapter.
