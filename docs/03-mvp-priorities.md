# Этап 3. MVP Приоритеты и Матрица функций

## 1. Матрица «Ценность / Сложность» (Value vs Complexity)

| Функция | Ценность для пользователя | Сложность реализации | Комментарий |
| :--- | :--- | :--- | :--- |
| **Поиск вратаря (SOS-кнопка)** | Высокая | Низкая | Базовый чат/уведомления по фильтру "Вратарь" |
| **Автоматизированный сбор денег** | Высокая | Средняя | Требует интеграции с платежными шлюзами или API банков |
| **Интеграция с порталами аренды (API)** | Средняя | Высокая | API закрыты, данные нестабильны, нужно договариваться |
| **Единый профиль со статистикой лиг** | Высокая | Высокая | Требует парсинга множества сайтов (НХЛ, ЛХЛ и др.) |
| **Маркетплейс экипировки (новое)** | Низкая | Высокая | Огромная конкуренция с сетевыми магазинами |
| **Барахолка (C2C) б/у формы** | Средняя | Средняя | Нужна модерация и система доверия |
| **Рейтинг надежности игрока (Карма)** | Высокая | Низкая | Простая логика отзывов после игры |
| **Расписание тренировок и игр команды** | Высокая | Низкая | Календарь + push-уведомления |

## 2. Топ-5 функций для запуска MVP (Core Features)

1.  **Smart Roster Management (Умный состав):** Инструмент для капитана, чтобы видеть, кто идет на игру, а кто нет. Автоматические напоминания.
2.  **Goalkeeper SOS:** Специализированный раздел/уведомления для быстрого поиска вратарей (бесплатно или за оплату льда).
3.  **Unified Player Identity (Хоккейный паспорт):** Профиль с указанием амплуа, уровня (Дебютант/Любитель/Мастер) и района проживания.
4.  **Ice Finder (Light):** Интерактивная карта ледовых арен Москвы с актуальными контактами и ссылками на внешние системы бронирования (без глубокой интеграции на старте).
5.  **Game Feedback & Karma:** Возможность подтвердить участие игрока и поставить оценку за адекватность/уровень.

## 3. Главные риски интеграции

### Технические риски
- **Отсутствие API:** Большинство порталов аренды (НаЛёд и др.) не имеют публичных API. Парсинг может привести к блокировкам.
- **Неактуальность данных:** Свободный слот на портале может быть уже забронирован по телефону. Риск "двойных продаж".
- **Разнообразие форматов лиг:** У каждой лиги своя структура данных (JSON, HTML, PDF протоколы).

### Юридические риски
- **Персональные данные (ФЗ-152):** Хранение статистики игроков, их ФИО и фото требует строгого соблюдения законодательства РФ.
- **Авторские права на данные:** Лиги могут запрещать использование своей статистики в коммерческих целях без договора.

### Ограничения
- **Монополия:** Крупные магазины могут не давать фиды данных мелким стартапам.

## 4. Открытые вопросы (Open Questions)

1.  **Уровни игроков:** Как объективно разделить "Любитель 1" от "Любитель 5"? Будем ли мы использовать существующие классификации лиг (например, РТХЛ) или введем свою?
2.  **Авторизация:** Достаточно ли входа через Telegram/VK или нужна связка с Госуслугами для подтверждения личности в турнирах?
3.  **Монетизация:** На чем зарабатываем в первый год? Комиссия с транзакций, платное продвижение объявлений о доборе или подписка для капитанов?
4.  **API Арен:** Готовы ли мы платить агрегаторам за доступ к их данным, если они предложат платное API?
5.  **Вратари:** Должен ли сервис быть бесплатным для вратарей как для самого дефицитного ресурса?
